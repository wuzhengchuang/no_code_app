## 集成测试场景

1. **完整用户注册-登录-修改流程**: 新用户注册 → 登录获取 token → 获取用户信息 → 修改名称 → 修改密码 → 旧密码登录失败 → 新密码登录成功

2. **团队创建与成员管理流程**: 用户A登录创建团队 → 添加用户B为 member → 用户B登录查看团队 → 尝试删除团队失败 → 用户A修改用户B为 admin → 用户B添加用户C → 用户B移除用户C → 用户A删除团队

3. **并发 Token 刷新**: 登录获取双 token → 使用 refresh_token 刷新 access_token → 旧 access_token 仍有效 → 旧 refresh_token 无法再次使用
