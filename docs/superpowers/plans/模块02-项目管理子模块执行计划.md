## 集成测试场景

1. **完整项目生命周期**: 用户登录 → 创建项目 → 更新配置 → 创建快照 v1.0 → 继续修改 → 创建快照 v1.1 → 恢复到 v1.0 → 创建分享链接 → 访问分享 → 删除分享 → 删除项目

2. **团队项目协作**: 用户A创建团队项目 → 添加用户B为 admin → 用户B登录修改项目 → 创建快照 → 用户C尝试访问失败 → 用户A删除项目

3. **项目分享与复制**: 用户A创建项目 → 创建分享链接 (权限 copy, 密码保护) → 用户B不输入密码访问失败 → 输入密码成功 → 复制项目 → 修改不影响原项目 → 分享过期后无法访问

4. **项目导入导出**: 用户A创建项目配置 → 导出 JSON → 用户B导入 → 验证配置一致

## 完成总结

### 已实现内容

1. **数据库模型** (`src/models/`)
   - `project.py`: 项目表、页面表、页面状态表
   - `snapshot.py`: 项目快照表
   - `share.py`: 项目分享表
   - `template.py`: 项目模板表
   - `collaborator.py`: 项目协作者表
   - `component.py`: 组件模板和实例表

2. **Pydantic Schema** (`src/schemas/`)
   - `project.py`: 项目相关 Schema
   - `page.py`: 页面相关 Schema
   - `snapshot.py`: 快照相关 Schema
   - `share.py`: 分享相关 Schema
   - `template.py`: 模板相关 Schema
   - `collaborator.py`: 协作者相关 Schema

3. **服务层** (`src/services/`)
   - `project_service.py`: 项目 CRUD、权限检查、复制、导出导入
   - `snapshot_service.py`: 快照创建、恢复、删除
   - `share_service.py`: 分享链接创建、访问、复制
   - `template_service.py`: 模板管理
   - `collaborator_service.py`: 协作者管理
   - `export_service.py`: 项目导出
   - `import_service.py`: 项目导入

4. **API 路由** (`src/api/v1/`)
   - `projects.py`: 项目 CRUD、复制、导出导入
   - `snapshots.py`: 快照管理
   - `shares.py`: 分享管理
   - `templates.py`: 模板管理
   - `collaborators.py`: 协作者管理

### 关键实现细节

- 权限检查采用多层次验证: owner → user_project_permissions → team 权限 → public 项目
- 分享功能支持密码保护、过期时间、查看次数限制
- 快照版本号自动生成 (v1.0.0, v1.0.1, v1.1.0)
- 项目导出支持完整数据结构(JSON格式)
- 使用 SQLite 进行开发测试，支持 MySQL 生产环境
